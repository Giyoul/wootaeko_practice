```markdown
구입금액을 입력해 주세요.
5000

[40, 1, 42, 32, 37, 10]
[4, 44, 42, 1, 18, 13]
[39, 37, 10, 14, 43, 33]
[31, 34, 26, 44, 37, 16]
[28, 20, 26, 33, 9, 35]

1,4,18,2,3,9

20

6개 일치 (2,000,000,000원) - 0개
총 수익률은 100.0%입니다.
```

## 구현 기능 목록

### 1. 입출력 기능

- [x]  기능 선택 메시지를 출력한다.
- [x]  과정, 레벨, 미션 메시지를 출력한다.
- [x]  페어 매칭시 과정, 레벨, 미션 메시지를 출력한다.
- [ ]  페어 조회시 과정, 레벨, 미션 메시지를 출력한다.
- [ ]  페어 매칭 결과를 출력한다.
- [ ]  매칭 경우의 수가 없을 경우에 에러 메시지를 출력한다.
- [ ]  매칭정보가 있을 경우 재매칭 여부를 묻는 메시지를 출력한다.
- [ ]  매칭 이력이 없을 경우에 에러 메시지를 출력한다.
- [ ]  초기화 확인 메시지를 출력한다.
- [x]  기능 선택을 입력받는다.
- [ ]  유저에게 다시 매칭받을지 여부를 입력받는다.
- [ ]  

### 2. 핵심 로직 기능

- [ ]  페어 매칭시, 두명씩 pair로 매칭한다.
- [ ]  페어 매칭시, 홀수이면 마지막 팀을 세 명으로 매칭한다.
- [ ]  같은 level에서 만난 적 있는 짝이면 다시 섞는다.
- [ ]  3회동안 매칭이 안되거나, 경우의 수가 없으면 에러 메시지를 출력하고, 다시 입력받는다.
- [ ]  페어 초기화시에, 페어 정보를 전부 초기화한다.
- [ ]  

### 3. 예외 처리 기능

- [ ]  기능 선택시에 1, 2, 3, Q가 아닌 다른 값을 입력할 경우 IllegalArgumentException으로 에러 메시지를 출력하고 다시 입력 받는다.
- [ ]  다시 매칭할 것인지에 대한 답변이 네 / 아니오 가 아닐 경우 IllegalArgumentException으로 에러 메시지를 출력하고 다시 입력 받는다.
- [ ]  과정, 레벨, 미션을 잘못 입력할 경우 IllegalArgumentException으로 에러 메시지를 출력하고 다시 입력 받는다.

### 4. 기타
